# Task 20260403-020: Evaluator Harness For Package Readiness

**Architectural Authority**: The growing `feasibility-core` package and the need to assess whether it is actually ready for external use as a staffing-feasibility library.
**Constraint**: This task must evaluate the package as a product surface, not just as a code artifact.

## Objective

Create a lightweight evaluator harness or checklist that answers:

- what the package can currently model
- what it cannot yet model
- whether it is ready for early external users
- what the next highest-value gaps are

## Deliverables

### Step 1: Define Evaluation Criteria

- [ ] supported domain features
- [ ] explanation quality
- [ ] API clarity
- [ ] documentation completeness
- [ ] test realism

### Step 2: Evaluate Current Package State

- [ ] score or summarize current support across the criteria
- [ ] identify blockers for early external use
- [ ] identify misleading rough edges

### Step 3: Produce Readiness Artifact

- [ ] concise readiness report
- [ ] explicit supported / unsupported matrix
- [ ] recommended next milestone after current tasks land

## Acceptance Criteria

- [ ] the repo has a reusable readiness/evaluation artifact
- [ ] package capability boundaries are easier to communicate
- [ ] next priorities are informed by explicit evaluation rather than intuition alone
